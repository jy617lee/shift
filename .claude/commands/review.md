아래 순서로 PR 전 코드 리뷰를 진행해줘.

1. `git diff main...HEAD`로 변경된 코드를 확인한다.
2. `code-review-rules.md`의 각 항목을 기준으로 위반 사항을 찾는다.
3. 결과를 아래 형식으로 출력한다:

---
**[규칙 번호. 규칙 이름]**
- 파일명:줄번호 — 문제 설명
- 개선 방향 (1~2줄)

문제가 없으면 "이상 없음"으로 표시한다.

---

마지막에 전체 요약을 한 줄로 작성한다. (예: "총 3건 — 네이밍 1, 중복 1, null 안전성 1")
수정이 필요한 항목은 직접 고칠지 물어봐줘.
